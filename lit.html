Overview

  • Lit components are web components.

  • Shadow DOM - Custom components have their own isolated & encapsulated DOM

  • Component:

import {LitElement, html} from 'lit';
import {customElement} from 'lit/decorators.js';
                            
@customElement('my-element')
export class MyElement extends LitElement {
    render() {
        return html`
            <p>Hello world! From my-element.</p>
        `;
    }
}
  • Reactive Properties - @property or @state. Changing a reactive property triggers the component to update.

  • Decorators - @customElement, @property, @state, etc.

    • @property: public properties that are part of the component's API and accessible/controllable via attributes:
      • @property()
      message: string = 'Hello.';
       use this.message or ${this.message} in HTML return
    • @state: private internal state that should not be exposed to consumers of the component
    • @query: converts a class property into a getter that executes a querySelector on the element's renderRoot:
      @query('#newitem')
      input!: HTMLInputElement;
      So input is now a getter of type HTMLInputElement

  • Declarative event listener:

   <input @input=${this.changeName} placeholder="Enter your name">

"this.changeName" is a event handler method you can define, like:

changeName(event: Event) {
    const input = event.target as HTMLInputElement;
    this.name = input.value;
}

"@input" can be replaced with any event name and "this.changeName" with any JavaScript expression that evaluates to an event handler, like:

   <button @click=${this.handleClick}>Click me!</button>
  • Expressions - dynamically set HTML attributes, properties, event listeners such as above, etc.
    • Boolean attributes:
      html`<div ?hidden=${!show}></div>`
    • Properties:
      html`<input .value=${value}>`
      (which is the same as inputEl.value = this.itemCount;)
  • Encapsulated Styles - affects the component only by using the shadow DOM
    static styles = css`
      .completed {
        text-decoration-line: line-through;
      }
    `;

Attribute Binding (Expressions)

  • Dynamically set HTML attributes to react to changes in the component's properties or state
  • . - Property binding, such as html`<input .value=${value}>`
  • ? - Boolean attribute binding, such as html`<div ?hidden=${!show}></div>`
  • @ - Event binding, such as event listeners: <button @click=${this.handleClick}>

Event Handling

  • Use the @ prefix in your template to bind event listeners to DOM events:
    <button @click=${this.handleClick}>
  • Define methods in your component class to handle these events:
    handleClick() { ... }
  • Update component properties within event handlers to trigger re-rendering:
    @property()
    buttonText: string = 'Click me';
    ...
    handleClick() {
      this.buttonText = 'Clicked!';
    }

Slotting

  • Docs
  • Necessary for your component to render children
    • render() {
      return html`
        <div class="header">
          <slot name="header"></slot>
        </div>
        <div class="content">
          <slot>Fallback content if no children assigned to this slot.</slot>
        </div>
        <div class="footer">
          <slot name="footer"></slot>
        </div>
      `;
    }
    Usage:
    <my-element>
      <p>This will go in the unnamed slot.</p>
      <span slot="header">This will go into the header slot.</span>
      <span slot="footer">This will go into the footer slot.</span>
    </my-element>
  • Accessing slotted children:
    • Use slot.assignedNodes or slot.assignedElements methods with the slotchange event
    • Example getter:
      get _slottedChildren() {
        const slot = this.shadowRoot.querySelector('slot');
        return slot.assignedElements({flatten: true});
      }
    • Call an event if slot changes:
      handleSlotchange(e) {
        const childNodes = e.target.assignedNodes({flatten: true});
      }
      render() {
        return html`<slot @slotchange=${this.handleSlotchange}></slot>`;
      }
